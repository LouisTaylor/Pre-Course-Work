HTML

  • All HTML code must start with !DOCTYPE html
  • All code has the < and > symbols
  • Head element contains information for the web browsers but not shown to a user
  • the body element reprents what is show on the browser

CSS

  • Inline CSS is where I can add CSS code directly to HTML code by simply adding it in
  • There are internal CSS spreadsheets

    • This a code added directly to the head of the webpage. It'll inform this page only that all corresponding elements should be a certain critera

  • External CSS style sheet
  • CSS uses { and } symbols for code
  • A period before a will select a class
  • I can define an element with a class giving it a specific id such as "class="p1"" to separate it from other similar elements
  • Margins and padding are different!

    • Padding is the space between content and the border, margins is the space between the border and the next element (see picture below)

Git

  • First, hard to navigate

    • I find the git codes not to be intuitive, best to work on pull/push requests and practice the codes

  • "git status" checks what branch we are currently on
  • "git checkout -b branch-name" creates a new branch
  • Should pull main to make sure it is up to date first by "git pull origin main"

    • This is just to help make sure that I have the most complete version if it the main was updated while I was working

  • "git push origin "name of branch" "

    • This will then push my finialized branch edits to the main code in the repository

  • "git add -A" adds all changes
  • "git commit -m "description" " adds details to what was changed

JavaScript

  • Every code of javascript ends in a ;
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Boolean is for true/false/null statements and array has multiple entries for the variable
